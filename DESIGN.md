# LIFE OS v2 詳細設計書

## 1. システムアーキテクチャ

LIFE OS v2 は、Notion APIとGoogle Calendar APIを組み合わせ、毎日のコグニティブ（認知）負荷を最小限に抑えるように設計されています。ユーザーは複雑なデータベース構造を意識することなく、毎日の「デイリーノート」という1枚のキャンバスだけで作業を完結できます。

```
+-------------------------------------------------------------+
|                        🧠 LIFE OS                           |
|                    (ホームダッシュボード)                   |
+------------------------------+------------------------------+
                               |
                               v
+-------------------------------------------------------------+
|                      📅 DAILY DB                            |
|                  (デイリーノートの実体)                     |
+------------------------------+------------------------------+
                               |
        +----------------------+----------------------+
        |                                             |
        v                                             v
+------------------------------+             +----------------+
|          ✅ TASK DB          |             | 🚀 PROJECT DB  |
|     (タスクの蓄積・状態管理) |             | (PJの状態・進捗)|
+------------------------------+             +----------------+
```

---

## 2. コグニティブ設計（GTD × バレットジャーナル）

本システムは、以下の2つの時間管理・思考整理思想をデジタルに落とし込んでいます。

### GTD (Getting Things Done) の実装
- **収集 (Capture)**: デイリーノートの `LOG` セクションに、ミーティングメモ、思ったこと、感情をすべてベタ打ちします。
- **処理 (Clarify)**: 毎朝7:00にSORAが前日の `TODAY` セクションをスキャンし、チェックボックス付きの項目を自動で `TASK DB` に登録します。夜間スキャンは廃止し、朝の処理に統合してクレジット消費を最小化しています。
- **整理 (Organize)**: タスクは自動的にプロジェクトや日付に紐付けられます。

### バレットジャーナルの実装
- **デイリーログ**: 毎日1ページ、その日の日付タイトル（例: `2026-05-30（土）`）のページが自動生成されます。
- **移行 (Migration)**: 前日に完了しなかったタスクは、翌朝7:00に自動的に新しいデイリーノートの `TODAY` セクションに「🔁」マーク付きで引き継がれます。これにより、タスクの「やり忘れ」を防ぎます。

## 3. スクリプトの実行環境と設定

自動化スクリプトは、Linux環境のcronまたはManusのスケジュール機能を利用して実行されます。

### 必要な環境変数・設定
スクリプトを実行する環境には、以下の設定が必要です。

```bash
# Notion API 設定
NOTION_API_KEY="<your_notion_api_key>"

# データベースID設定
DAILY_DS_ID="f9b89321-f903-4167-b022-0787096ea6f3"
TASK_DS_ID="47ae00b4-956c-4fa4-a786-af39a5b33067"
PROJ_DS_ID="eadd59d6-f0d5-4356-bc13-76aeab13c0ec"
```

> ※ パスやAPIキーなどの個人固有情報は、実行環境に合わせて `<ユーザー名>` などの変数表記に差し替えて設定してください。
