# LIFE OS v01 — TAKUMI's Cognitive Workspace

LIFE OS v01 は、Notionを唯一の操作起点とし、GTD（Getting Things Done）とバレットジャーナル（Bullet Journal）の思想を融合させた、TAKUMI専用の日常・ビジネス統合型コグニティブワークスペースです。

毎朝7:00に自動でデイリーノートが生成され、その日の予定・タスク・プロジェクト状況がすべて1ページに集約されます。前日の未完了タスクの引き継ぎ（Migration）と、TASK DBへの同期も朝の処理に統合されています。

---

## システムの特徴

**唯一の操作起点（デイリーノート）**

毎日1ページだけ開けば、その日のすべて（予定・タスク・PJ・メモ）が完結します。複雑なDBを直接操作する必要はありません。

**GTD × バレットジャーナル**

`Capture`（LOGに何でも書く）→ `Clarify`（翌朝SORAがTASK DBに自動登録）→ `Engage`（TODAYセクションで実行）というGTDの流れを、バレットジャーナルのMigration（未完了タスクの翌日引き継ぎ）と組み合わせて自動化しています。

**朝7:00の1回実行でクレジット消費を最小化**

夜間スキャンは廃止し、前日スキャン・TASK DB同期・当日デイリー生成をすべて朝7:00の1回の実行に統合しています。

---

## 自動化フロー（毎朝7:00 JST）

```
[0] 前日のデイリーTODAYをスキャン → TASK DBに同期
    チェック済み → STATUS=DONE で登録
    未チェック  → STATUS=TODO, MIGRATED=false で登録

[1] TASK DBから未完了タスクを取得（翌日への引き継ぎ）

[2] PROJECT DBからACTIVEプロジェクトの進捗・タスクを取得

[3] Googleカレンダーから当日イベントを取得
    ミーティングイベント → サブページを自動生成（AI Meeting Note欄付き）

[4] DAILY DBに当日のデイリーノートを作成

[5] ミーティングサブページをデイリーノート配下に作成

[6] 本文コンテンツを生成してページを更新
    金曜日のみ WEEKLY REVIEW セクションを追加
```

## デイリーノートの構造

```
📅 2026-05-30（金）

## 📅 SCHEDULE
  10:00  [○○ミーティング](ミーティングサブページへのリンク)
  14:00  △△

## ✅ TODAY
  🔁 [ ] 前日から引き継いだタスク（TASK DBから）
  [PJ: ○○] [ ] プロジェクトタスク（PROJECT DBから）
  [ ] （手動追加可能）

## 🚀 PROJECT STATUS
  ○○プロジェクト  40%  → 次: ○○

## 📝 LOG
  （自由記述。ミーティングメモ・気づき・感情・何でも）

## 🌙 WEEKLY REVIEW  ← 金曜のみ
  今週どうだった？ 感情・気づき・来週のテーマ
```
