# LIFE OS v2 データベーススキーマ仕様

LIFE OS v2 は、必要最小限の3つのリレーショナルデータベース（DAILY、TASK、PROJECT）で構成されています。

---

## 1. 📅 DAILY DB（デイリーノート）

毎日1ページ作成される、すべての情報の起点となるデータベースです。

| プロパティ名 | 型 | 用途・説明 |
|---|---|---|
| **DATE** | タイトル | ページのタイトル。`YYYY-MM-DD（曜日）` 形式。 |
| **DATE_RAW** | 日付 | フィルタリングやカレンダー表示に使用する日付型プロパティ。 |
| **TYPE** | セレクト | `Daily` または `Weekly Review`。 |
| **ENERGY** | 数値 | 1〜5の5段階評価。その日のエネルギーレベルを記録。 |
| **TAGS** | マルチセレクト | 自由なタグ付け用。 |

---

## 2. ✅ TASK DB（タスク）

すべてのタスク（今日やること、いつかやること、プロジェクトタスク）を蓄積・管理するデータベースです。

| プロパティ名 | 型 | 用途・説明 |
|---|---|---|
| **TASK** | タイトル | タスクの名前。 |
| **STATUS** | セレクト | `TODO` / `DOING` / `DONE` / `SOMEDAY`（いつかやる）。 |
| **PROJECT** | リレーション | 関連するプロジェクト（`PROJECT DB` へのリレーション）。 |
| **DAILY** | リレーション | タスクが作成された、または実行された日（`DAILY DB` へのリレーション）。 |
| **DUE** | 日付 | タスクの期限（任意）。 |
| **MIGRATED** | チェックボックス | 翌日に引き継ぎ処理が行われたかどうかのフラグ。 |

---

## 3. 🚀 PROJECT DB（プロジェクト）

進行中のプロジェクト、イベント、目標を管理するデータベースです。

| プロパティ名 | 型 | 用途・説明 |
|---|---|---|
| **PROJECT** | タイトル | プロジェクト名。 |
| **STATUS** | セレクト | `ACTIVE`（進行中） / `WAITING` / `DONE`（完了）。 |
| **PROGRESS** | ロールアップ | 紐付いている `TASK DB` の完了率（%）を自動計算。 |
| **NEXT_ACTION** | テキスト | 次に取るべき具体的なアクション（手動記入）。 |
| **GOAL** | テキスト | プロジェクトの目標・概要。 |
| **DUE** | 日付 | プロジェクトの期限（任意）。 |
| **TASKS** | リレーション | プロジェクトに紐付くタスク一覧（`TASK DB` への逆リレーション）。 |
